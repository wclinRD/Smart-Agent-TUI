Anche se Smart puo essere eseguito direttamente su Windows, consigliamo di usare [Windows Subsystem for Linux (WSL)](https://learn.microsoft.com/en-us/windows/wsl/install) per la migliore esperienza. WSL fornisce un ambiente Linux che funziona in modo ottimale con le funzionalita di Smart.

:::tip[Perche WSL?]
WSL offre prestazioni migliori del file system, supporto completo del terminale e compatibilita con gli strumenti di sviluppo su cui Smart fa affidamento.
:::

---

## Configurazione

<Steps>

1. **Installa WSL**

   Se non l'hai ancora fatto, [installa WSL](https://learn.microsoft.com/en-us/windows/wsl/install) seguendo la guida ufficiale Microsoft.

2. **Installa Smart in WSL**

   Dopo aver configurato WSL, apri il terminale WSL e installa Smart usando uno dei [metodi di installazione](/docs/).

   ```bash
   curl -fsSL https://smart.ai/install | bash
   ```

3. **Usa Smart da WSL**

   Vai nella directory del progetto (accedi ai file Windows tramite `/mnt/c/`, `/mnt/d/`, ecc.) ed esegui Smart.

   ```bash
   cd /mnt/c/Users/YourName/project
   smart
   ```

</Steps>

## App desktop + server WSL

Se preferisci usare l'app desktop di Smart ma vuoi eseguire il server in WSL:

1. **Avvia il server in WSL** con `--hostname 0.0.0.0` per consentire connessioni esterne:

   ```bash
   smart serve --hostname 0.0.0.0 --port 4096
   ```

2. **Collega l'app desktop** a `http://localhost:4096`

:::note
Se `localhost` non funziona nella tua configurazione, usa l'indirizzo IP di WSL (da WSL: `hostname -I`) e collega `http://<wsl-ip>:4096`.
:::

:::caution
Quando usi `--hostname 0.0.0.0`, imposta `SMART_SERVER_PASSWORD` per proteggere il server.

```bash
SMART_SERVER_PASSWORD=your-password smart serve --hostname 0.0.0.0
```

:::

## Client web + WSL

Per la migliore esperienza web su Windows:

1. **Esegui `smart web` nel terminale WSL** invece che in PowerShell:

   ```bash
   smart web --hostname 0.0.0.0
   ```

2. **Accedi dal browser Windows** su `http://localhost:<port>` (Smart stampa l'URL)

Eseguire `smart web` da WSL garantisce accesso corretto al file system e integrazione del terminale, restando comunque accessibile dal browser Windows.

---

## Accesso ai file Windows

WSL puo accedere a tutti i file Windows tramite la directory `/mnt/`:

- Unita `C:` → `/mnt/c/`
- Unita `D:` → `/mnt/d/`
- E cosi via...

Esempio:

```bash
cd /mnt/c/Users/YourName/Documents/project
smart
```

:::tip
Per un'esperienza piu fluida, valuta di clonare o copiare il repository nel file system di WSL (per esempio in `~/code/`) ed eseguire Smart li.
:::

---

## Suggerimenti

- Tieni Smart in esecuzione su WSL per i progetti salvati su unita Windows: l'accesso ai file e immediato
- Usa l'[estensione WSL di VS Code](https://code.visualstudio.com/docs/remote/wsl) insieme a Smart per un flusso di lavoro integrato
- Configurazione e sessioni di Smart sono salvate nell'ambiente WSL in `~/.local/share/smart/`
